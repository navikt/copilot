Mars 2026 markerer en uke der GitHub slapp ni Copilot-oppdateringer på fem dager. Fellesnevneren: agenter som jobber mer selvstendig, med bedre modeller, bedre kontekst og bedre innsyn for de som styrer plattformen.

## 1. GPT-5.4 er generelt tilgjengelig

OpenAI lanserte GPT-5.4 den 5. mars, og GitHub rullet den ut i Copilot samme dag. Modellen er tilgjengelig for Pro, Pro+, Business og Enterprise — i VS Code, Visual Studio, JetBrains, Xcode, Eclipse, GitHub.com, GitHub Mobile og CLI.

Det som skiller GPT-5.4 fra forgjengerne er at den er bygget for agentisk arbeid. Den kan planlegge flertrinnsoppgaver, kalle verktøy, vedlikeholde tilstand på tvers av en økt, og koordinere endringer på tvers av filer. I GitHubs tidlige testing viser den høyere suksessrate på realistiske utvikleroppgaver sammenlignet med GPT-5.2-Codex.

Kontekstvinduet er oppgradert til 1 million tokens (opp fra 400K). For Enterprise- og Business-kunder må administratorer aktivere GPT-5.4-policyen i Copilot-innstillingene før brukerne får tilgang.

**Kilde:** [GPT-5.4 is generally available in GitHub Copilot](https://github.blog/changelog/2026-03-05-gpt-5-4-is-generally-available-in-github-copilot/) (GitHub Changelog, 5. mars 2026)

## 2. Code review med agentisk arkitektur

Copilot code review er skrevet om med en agentisk verktøykallende arkitektur og er nå generelt tilgjengelig for alle betalende planer. Tidligere leste code review bare diffen. Nå navigerer den i repoet — henter relevant kode, katalogstruktur og referanser — for å forstå hvordan endringene passer inn i helheten.

Resultatet ifølge GitHub: høyere kvalitet på funn, mindre støy, og mer handlingsrettede anbefalinger. I 71 % av gjennomgangene gir den konkret tilbakemelding. I 29 % sier den ingenting — fordi stillhet er bedre enn støy. Snitt er 5,1 kommentarer per review.

Et interessant designvalg: da de byttet til en bedre resonnerende modell, økte positiv tilbakemelding med 6 %, selv om ventetiden økte 16 %. GitHub valgte bevisst signal over hastighet. Code review bruker nå også Copilot Memory, slik at mønstre oppdaget i én PR kan gjenbrukes i neste.

Agentisk code review kjører på GitHub Actions. Team med self-hosted runners trenger ett engangsoppsett.

## 3. Copilot coding agent for Jira

Du kan nå tilordne Jira-saker til Copilot coding agent direkte fra Jira. Agenten leser oppgavekonteksten, jobber i GitHub-repoet, og leverer en draft-PR. Funksjonen er i public preview.

Flyten: installer GitHub Copilot for Jira-appen fra Atlassian Marketplace, koble til GitHub-organisasjonen, og sett «GitHub Copilot» som ansvarlig på en Jira-sak. Agenten starter automatisk. Koden gjennomgås og merges via vanlige PR-regler — ingen automatisk merging.

For team som planlegger i Jira men bygger i GitHub, fjerner dette et kontekstbytte. Krever Jira Cloud med Rovo aktivert og Copilot coding agent innslått.

**Kilde:** [GitHub Copilot coding agent for Jira is now in public preview](https://github.blog/changelog/2026-03-05-github-copilot-coding-agent-for-jira-is-now-in-public-preview/) (GitHub Changelog, 5. mars 2026)

## 4. Grok Code Fast 1 — xAI inn i Copilot

xAI lanserte **Grok Code Fast 1**, en ny modell bygget fra bunnen av for agentisk koding. Den er optimalisert for verktøybruk (grep, terminal, fildrifting) og bruker en mixture-of-experts-arkitektur. Ifølge xAI er den «sannsynligvis den raskeste modellen der ute akkurat nå», med cache hit rates over 90 %.

For Copilot Free-brukere er Grok Code Fast 1 tilgjengelig via auto model selection — Copilot velger modellen automatisk. For betalende planer er den en valgbar modell i model picker. xAI tilbyr modellen gratis for en begrenset periode gjennom utvalgte partnere, inkludert GitHub Copilot, Cursor, Cline og Windsurf.

Dette betyr at Copilot nå har modeller fra fire leverandører: OpenAI, Anthropic, Google og xAI. Konkurransen om å være standardmodellen i utviklerens IDE er reell.

## 5. Memory på som standard

Copilot Memory er nå aktivert som standard for alle Pro- og Pro+-brukere. Tidligere var det opt-in. Memory lar Copilot lagre repo-spesifikke fakta som deles mellom coding agent, code review og CLI. Minner utløper etter 28 dager og er knyttet til repoet.

For organisasjoner er Memory fortsatt slått av som standard og må aktiveres av administratorer. Hvis en bruker har abonnement fra flere organisasjoner, gjelder den mest restriktive innstillingen.

Endringen signalerer at GitHub ser Memory som klar nok til å bli defaults-opplevelsen — ikke bare et eksperiment for early adopters.

**Kilde:** [Copilot Memory now on by default for Pro and Pro+ users in public preview](https://github.blog/changelog/2026-03-04-copilot-memory-now-on-by-default-for-pro-and-pro-users-in-public-preview/) (GitHub Changelog, 4. mars 2026)

## 6. Enterprise-kontrollplan og agentstyring

Enterprise AI Controls og agent control plane er nå generelt tilgjengelig. Det gir organisasjoner en samlet flate for å styre AI-policyer, custom agents og agenters aktivitet.

Nytt i mars:

- **Sesjonsfiltre**: søk etter agenters aktivitet filtrert på status, repo, bruker og agent (inkludert tredjepartsagenter)
- **Custom agents via API**: definer og administrer enterprise-brede agenter programmatisk
- **Revisjonslogging**: alle agenthandlinger logges med `actor_is_agent`-identifikator og hvem agenten handler på vegne av
- **Push-regler for agentfiler**: beskytt `.github/agents/*.md`-stier med ett klikk, slik at bare autoriserte endrer agentdefinisjoner
- **Modellvelger i PR-kommentarer**: velg hvilken modell som svarer når du nevner @copilot i en PR

MCP enterprise-allowlists forblir i preview. GitHub designer en løsning som skalerer bedre på tvers av organisasjoner uten å kreve flere registre.

## 9. Fra utviklermiljøet

Stemningen i mars 2026 er todelt: begeistringen for agentiske arbeidsflyter kolliderer med praktisk realisme.

**Det som fungerer**: En tilbakevendende observasjon på Hacker News er at «den beste programvaren for en agent er det som er best for en programmør.» Gode spesifikasjoner, klare tester og strukturert kodebase gir agenter et solid grunnlag. Folk som lykkes med kodingsagenter tvinger seg til å artikulere krav med maskinpresisjon — og finner at dette i seg selv forbedrer utviklingsprosessen.

**Det som bekymrer**: En juniorutvikler på Reddit skrev: «Noen ganger aksepterer jeg AI-kode uten å forstå den fullt ut.» Linus Torvalds har advart mot å blindt stole på AI i seriøse, langsiktige prosjekter. En Hacker News-diskusjon om rammeverk vs. agenter påpekte at agenter «kutter hjørner du aldri visste ble kuttet», og at dette fører til subtile bugs som dukker opp langt senere.

Addy Osmani (Google) oppsummerte trenden: «Orchestration > Assistance» — utviklere blir mer som flåtekommandanter enn kodeskrivere. Men han observerte også at enterprise-AI fortsatt bryter ned når beslutninger må være «reproduserbare, reviderbare og trygge nok til at noen er villig til å signere under dem.»
