Mars 2026 var uka da GitHub slapp ni Copilot-oppdateringer på fem dager — og fulgte opp med enda flere den neste. Fellesnevneren: agenter som jobber mer selvstendig, med bedre modeller, bedre kontekst og bedre innsyn for de som styrer plattformen.

## 1. Code review med agentisk arkitektur

Copilot code review er skrevet om med en agentisk arkitektur og er nå GA for alle betalende planer. Tidligere leste den bare diffen. Nå navigerer den i repoet — henter relevant kode, katalogstruktur og referanser — for å forstå hvordan endringene passer inn i helheten.

I 71 % av gjennomgangene gir den konkret tilbakemelding. I 29 % sier den ingenting — fordi stillhet er bedre enn støy. Da GitHub byttet til en bedre resonnerende modell, økte positiv tilbakemelding med 6 %, selv om ventetiden økte 16 %. De valgte bevisst signal over hastighet.

Code review bruker nå Copilot Memory, slik at mønstre oppdaget i én PR gjenbrukes i neste. Du kan også be om review fra terminalen med `gh pr edit --add-reviewer @copilot`.

Kjører på GitHub Actions. Team med self-hosted runners trenger ett engangsoppsett.

## 2. GPT-5.4 er generelt tilgjengelig

OpenAI lanserte GPT-5.4 den 5. mars, og GitHub rullet den ut i Copilot samme dag. Modellen er bygget for agentisk arbeid — flertrinnsplanlegging, verktøykall og tilstandshåndtering på tvers av økter. Kontekstvinduet er oppgradert til 1 million tokens (opp fra 400K).

For Business- og Enterprise-kunder må administratorer aktivere GPT-5.4-policyen i Copilot-innstillingene før brukerne får tilgang. Copilot har nå modeller fra fire leverandører (OpenAI, Anthropic, Google, xAI) etter at xAIs **Grok Code Fast 1** også ble tilgjengelig i mars.

**Kilde:** [GPT-5.4 is generally available in GitHub Copilot](https://github.blog/changelog/2026-03-05-gpt-5-4-is-generally-available-in-github-copilot/) (GitHub Changelog, 5. mars 2026)

## 3. VS Code v1.110 — hooks, plugins og nettleserverktøy

Februarreleasen av Copilot i VS Code gjør agenter mer praktiske for lengre oppgaver. Det viktigste:

- **Hooks**: kjør egen kode ved agentlivssyklushendelser — håndhev policyer, kjør auto-lint, eller blokker kommandoer
- **Agent-plugins** (eksperimentell): installer ferdigpakkede bunter av skills, verktøy, hooks og MCP-servere
- **Agentisk nettleserverktøy** (eksperimentell): la agenter drive den integrerte nettleseren for å verifisere egne endringer
- **Delt Memory**: kunnskap deles på tvers av coding agent, CLI og code review
- **Innebygd Explore-subagent**: delegerer parallellisert kodebaseforskning til lette modeller
- **Forgrening av samtaler** og **kø fra chat**: start alternative grener fra checkpoints, send oppfølgingsmeldinger mens agenten jobber

**Kilde:** [GitHub Copilot in Visual Studio Code v1.110 – February release](https://github.blog/changelog/2026-03-06-github-copilot-in-visual-studio-code-v1-110-february-release/) (GitHub Changelog, 6. mars 2026)

## 8. GPT-5.3-Codex får langtidsstøtte

GitHub introduserer et nytt konsept: langtidsstøtte (LTS) for AI-modeller. GPT-5.3-Codex er første modell ut, med garantert 12 måneders tilgjengelighet for Business- og Enterprise-kunder — fra lansering 5. februar 2026 til 4. februar 2027. Bakgrunnen er at enterprise-kunder trenger tid til interne sikkerhets- og risikovurderinger før de godkjenner nye modeller.

Samtidig erstatter GPT-5.3-Codex den eldre GPT-4.1 som standardmodell. Organisasjoner som ikke har godkjent andre modeller ennå, vil automatisk bli migrert til GPT-5.3-Codex innen 60 dager. GPT-4.1 forblir tilgjengelig med 0x premium request-multiplikator inntil videre, mens GPT-5.3-Codex har 1x-multiplikator.

Data fra GitHub viser at GPT-5.3-Codex har høy «code survival rate» — kode som foreslås av modellen overlever i produksjon over tid.

**Kilde:** [GPT-5.3-Codex long-term support in GitHub Copilot](https://github.blog/changelog/2026-03-18-gpt-5-3-codex-long-term-support-in-github-copilot) (GitHub Changelog, 18. mars 2026)

## 9. Konfigurerbare valideringsverktøy for coding agent

Coding agent kjører automatisk CodeQL, GitHub Advisory Database, secret scanning og Copilot code review som en del av valideringen. Disse verktøyene er gratis, krever ikke GitHub Advanced Security-lisens, og er aktivert som standard.

Men ikke alle team har nytte av alle sjekkene. For eksempel kan CodeQL-analyse ta lang tid for store prosjekter og forsinke feedback-loopen. Repository-administratorer kan nå velge hvilke valideringsverktøy coding agent skal kjøre, via **Copilot → Coding agent** i repository-innstillingene.

**Kilde:** [Configure Copilot coding agent's validation tools](https://github.blog/changelog/2026-03-18-configure-copilot-coding-agents-validation-tools) (GitHub Changelog, 18. mars 2026)

## 10. Coding agent starter 50 % raskere

GitHub har optimalisert oppstartstiden for coding agent, som nå starter arbeidet 50 % raskere enn før. Dette påvirker alle måter å starte agenten på — tildeling av issues til Copilot, prompts i Agents-fanen, og `@copilot`-nevnelser i PR-kommentarer.

Raskere oppstart betyr kortere tid til første PR, og en strammere feedback-loop når du itererer med `@copilot` på en eksisterende PR. Kombinert med den nye konfigurasjonen av valideringsverktøy (seksjon 9) kan team som prioriterer hastighet over fullstendig validering få betydelig raskere sykluser.

**Kilde:** [Copilot coding agent now starts work 50% faster](https://github.blog/changelog/2026-03-19-copilot-coding-agent-now-starts-work-50-faster) (GitHub Changelog, 19. mars 2026)

## 11. Bedre innsyn i coding agent-sesjoner

Sessjonsloggene for coding agent har fått tre forbedringer som gjør det enklere å forstå hva agenten gjør:

- **Innebygde setup-steg**: Loggene viser nå når repoet klones og agent firewall startes, så du slipper å lure på hva som skjer under oppstart.
- **Custom setup-output**: Hvis repoet har en `copilot-setup-steps.yml`, vises output direkte i sessjonsloggen — du slipper å navigere til GitHub Actions for feilsøking.
- **Subagent-aktivitet**: Når coding agent delegerer til subagenter (for eksempel for kodebaseforskning), vises aktiviteten kollapset med en heads-up display som forteller hva subagenten jobber med akkurat nå.

Forbedringene gir utviklere og team bedre oversikt uten å måtte dykke ned i Actions-logger.

**Kilde:** [More visibility into Copilot coding agent sessions](https://github.blog/changelog/2026-03-19-more-visibility-into-copilot-coding-agent-sessions) (GitHub Changelog, 19. mars 2026)

## 13. @copilot kan nå endre eksisterende PR-er direkte

Du kan nå nevne `@copilot` i en hvilken som helst pull request for å be coding agent gjøre endringer direkte. Tidligere åpnet Copilot en ny PR oppå den eksisterende — nå pusher agenten endringene rett til PR-branchen. Du kan be Copilot fikse feilende tester, adressere review-kommentarer, eller gjøre andre endringer med naturlig språk.

Denne endringen gjør at `@copilot` fungerer slik utviklere intuitivt forventer: som en kollega som pusher en fix til branchen din. Ønsker du fortsatt den gamle atferden med en separat PR, kan du spesifisere det eksplisitt: `@copilot open a PR to fix the failing tests`.

Coding agent er tilgjengelig for alle betalende Copilot-planer. Business- og Enterprise-brukere må ha coding agent aktivert av en administrator.

**Kilde:** [Ask @copilot to make changes to any pull request](https://github.blog/changelog/2026-03-24-ask-copilot-to-make-changes-to-any-pull-request) (GitHub Changelog, 24. mars 2026)

## 14. API-styring av coding agent-tilgang

Organisasjoner som vil styre coding agent-tilgang i stor skala kan nå gjøre det programmatisk via nye REST API-er i public preview. API-ene lar deg bestemme om coding agent er tilgjengelig for ingen, alle, eller utvalgte repoer — og legge til eller fjerne repoer fra tilgangslisten.

For store organisasjoner som Nav med hundrevis av repoer er dette en viktig forbedring. I stedet for manuell konfigurering per repo i UI-et kan tilgangsstyringen automatiseres og integreres i eksisterende governance-verktøy og onboarding-prosesser.

**Kilde:** [Manage Copilot coding agent repository access via the API](https://github.blog/changelog/2026-03-24-manage-copilot-coding-agent-repository-access-via-the-api) (GitHub Changelog, 24. mars 2026)

## 15. Coding agent løser merge-konflikter

Copilot coding agent kan nå løse merge-konflikter direkte i pull requests. Du nevner `@copilot` i en kommentar og skriver for eksempel `@copilot Merge in main and resolve the conflicts`. Agenten jobber i sitt eget skybaserte utviklingsmiljø, gjennomfører endringene, kjører bygg og tester, og pusher resultatet.

Dette er en naturlig utvidelse av funksjonaliteten som ble lansert 24. mars (seksjon 13), der `@copilot` fikk muligheten til å endre eksisterende PR-er. Merge-konflikter har tradisjonelt vært en manuell og tidkrevende oppgave, spesielt for langlevde feature-brancher. Med denne endringen kan utviklere delegere oppgaven til agenten og fokusere på kode-review i stedet.

**Kilde:** [Ask @copilot to resolve merge conflicts on pull requests](https://github.blog/changelog/2026-03-26-ask-copilot-to-resolve-merge-conflicts-on-pull-requests) (GitHub Changelog, 26. mars 2026)

## 16. Copilot for Jira — modellvalg, sporbarhet og Confluence-kontekst

GitHub Copilot coding agent for Jira, som ble lansert i public preview 5. mars, har fått flere forbedringer basert på tilbakemeldinger fra tidlige brukere. De viktigste nyhetene:

- **Modellvalg**: Du kan nå velge hvilken AI-modell coding agent bruker, direkte fra Jira-kommentaren.
- **Sporbarhet**: Agenten inkluderer nå Jira-saksnummer i PR-tittel og grennavn, med lenke tilbake til den opprinnelige Jira-saken.
- **Confluence-kontekst via MCP**: Ved å konfigurere Atlassians MCP-server med en PAT kan coding agent hente designdokumenter og spesifikasjoner fra Confluence mens den jobber med Jira-saker.

Confluence-integrasjonen er et interessant eksempel på hvordan MCP utvider coding agents kontekst utover selve kodebasen. I stedet for å skrive lange beskrivelser i Jira-saken kan agenten selv hente relevant dokumentasjon.

**Kilde:** [GitHub Copilot for Jira — Public preview enhancements](https://github.blog/changelog/2026-03-25-github-copilot-for-jira-public-preview-enhancements) (GitHub Changelog, 25. mars 2026)

## 17. Bruksmetrikker skiller mellom IDE og coding agent

Copilot-bruksmetrikker har fått et nytt felt: `used_copilot_coding_agent`. Enterprise- og organisasjonsadministratorer kan nå se hvilke brukere som aktivt bruker coding agent — enten via tildeling av issues til Copilot eller ved å tagge `@copilot` i PR-kommentarer.

Tidligere var det vanskelig å skille mellom IDE-basert agent mode (`used_agent`) og den skybaserte coding agent. Med denne oppdateringen kan administratorer spore adopsjon av coding agent separat og få bedre innsikt i hvordan team bruker Copilot utover IDE-en. API-feltet er tilgjengelig i daglige og 28-dagers rapporter.

**Kilde:** [Copilot usage metrics now identify active Copilot coding agent users](https://github.blog/changelog/2026-03-25-copilot-usage-metrics-now-identify-active-copilot-coding-agent-users) (GitHub Changelog, 25. mars 2026)

## 18. Personvern og vilkårsendringer for AI-trening

GitHub har oppdatert personvernerklæringen og brukervilkårene med virkning fra 24. april 2026. Den viktigste endringen: interaksjonsdata fra Copilot Free, Pro og Pro+ — inkludert prompts, forslag og kodekontekst — kan brukes til å trene og forbedre AI-modeller med mindre brukeren reserverer seg.

Copilot Business og Enterprise er **ikke** berørt av endringen. For disse planene gjelder fortsatt Data Protection Agreement, og data brukes ikke til modelltrening. GitHub presiserer også at de ikke deler inputs eller outputs med tredjeparts AI-leverandører for selvstendig trening, og at innholdet i private repoer lagret «at rest» ikke brukes.

Oppdateringen inkluderer nye definisjoner i brukervilkårene (AI Feature, Input, Output) og en ny seksjon J som samler alle AI-relaterte vilkår. For brukere i EØS og Storbritannia oppgir GitHub «legitimate interest» som rettslig grunnlag.

**Kilde:** [Updates to our Privacy Statement and Terms of Service: How we use your data](https://github.blog/changelog/2026-03-25-updates-to-our-privacy-statement-and-terms-of-service-how-we-use-your-data) (GitHub Changelog, 25. mars 2026)

## 19. Gemini 3 Pro er utfaset

GitHub har utfaset Gemini 3 Pro fra alle Copilot-opplevelser — chat, inline edits, agent mode og kodekomplettering. Anbefalt erstatning er Gemini 3.1 Pro som ble tilgjengelig 23. mars (seksjon 12). Enterprise-administratorer som har aktivert Gemini 3 Pro-policyen trenger ikke gjøre noe for å fjerne modellen, men bør verifisere at Gemini 3.1 Pro er aktivert som alternativ.

**Kilde:** [Gemini 3 Pro deprecated](https://github.blog/changelog/2026-03-26-gemini-3-pro-deprecated) (GitHub Changelog, 26. mars 2026)
