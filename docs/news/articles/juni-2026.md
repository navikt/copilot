Microsoft Build 2026 ga en massiv Copilot-oppdatering 2. juni. Temaene er tydelige: agentisk utvikling modnes med sandkasser og automatiseringer, SDK-et går GA for egne integrasjoner, Copilot-appen får canvases som nytt interaksjonsmønster, og code review blir tilpassbart med MCP og skills. For Nav betyr dette at verktøyene for enterprise-styrt AI-utvikling tar et stort steg fremover.

---

## 1. Copilot SDK er generelt tilgjengelig

GitHub Copilot SDK går fra public preview til GA. SDK-et gir programmatisk tilgang til den samme agent-runtimen som driver Copilot — planlegging, verktøyinvokasjoner, filendringer og multi-turn-sesjoner.

SDK-et er nå tilgjengelig i seks språk: TypeScript, Python, Go, .NET, Rust (ny) og Java (ny). Nøkkelfunksjoner inkluderer custom tools og MCP-integrasjon, finjustering av system-promptet uten å skrive det fra scratch, OpenTelemetry-tracing med W3C trace context, og fleksibel autentisering via GitHub OAuth, GitHub Apps, miljøtokens og BYOK.

Siden public preview har SDK-et blitt brukt til CI/CD-assistenter, interne utviklerverktøy og kundevendte AI-funksjoner. API-overflaten er nå stabil og produksjonsklar.

**Kilde:** [Copilot SDK is now generally available](https://github.blog/changelog/2026-06-02-copilot-sdk-is-now-generally-available) (GitHub Changelog, 2. juni 2026)

## 2. Cloud- og lokale sandkasser i public preview

Copilot kan nå kjøre verktøy i isolerte sandkasser — både lokalt på utviklerens maskin og i skyen. Dette gir sikre, isolerte miljøer for agentens verktøykjøring uten at den har direkte tilgang til vertssystemet.

Lokale sandkasser isolerer filsystem og prosesser mens agenten jobber. Cloud-sandkasser gir fullverdige utviklingsmiljøer i skyen. Begge variantene er designet for å gjøre agentisk utvikling tryggere, spesielt når agenten installerer avhengigheter, kjører tester eller manipulerer filer.

**Kilde:** [Cloud and local sandboxes for GitHub Copilot now in public preview](https://github.blog/changelog/2026-06-02-cloud-and-local-sandboxes-for-github-copilot-now-in-public-preview) (GitHub Changelog, 2. juni 2026)

## 3. Copilot-appen: canvases og utvidet tilgang

Copilot-appen er nå tilgjengelig for alle Pro, Pro+, Business og Enterprise-kunder. Den store nyheten i denne utgivelsen er canvases — interaktive arbeidsflater der mennesker og agenter samarbeider.

Et canvas er en strukturert, interaktiv overflate over et arbeidsobjekt — en plan, PR, nettleser-sesjon, terminal, eller sjekkliste. Tre deltakere deler et canvas: brukeren inspiserer og styrer, agenten oppdaterer og bruker det som bevis på fremdrift, og appen kobler det til underliggende artefakter. GitHub kaller dette «agent experience» (AX).

Andre nyheter: stemmesamtaler med lokal tale-til-tekst, cloud-sesjoner direkte fra appen, planlagte automatiseringer, agentisk nettlesing, og Copilot CLI-sesjoner synlige i appens «My work»-visning.

**Kilde:** [Expanded technical preview availability for the GitHub Copilot app](https://github.blog/changelog/2026-06-02-expanded-technical-preview-availability-for-the-github-copilot-app) (GitHub Changelog, 2. juni 2026)

## 4. Planlagte automatiseringer for cloud agent

Cloud agent kan nå kjøre automatisk — på tidsplan eller som respons på repository-hendelser. Med automatiseringer kan du overlevere repetitive oppgaver til agenten uten at maskinen din trenger å være på.

Eksempler på bruk: daglige dependency-oppdateringer, automatisk triage av nye issues, periodisk kode-opprydding, eller respons på spesifikke hendelser som nye PR-er eller feilede builds.

**Kilde:** [Schedule and automate tasks with Copilot cloud agent](https://github.blog/changelog/2026-06-02-schedule-and-automate-tasks-with-copilot-cloud-agent) (GitHub Changelog, 2. juni 2026)

## 5. Code review med agent skills og MCP

Copilot code review kan nå tilpasses teamets verktøy og standarder. To nye funksjoner i public preview: agent skills lar code review kjøre teamets egne verktøy (lintere, type-checkere, sikkerhetsskannere) som del av gjennomgangen, og MCP-integrasjon lar den koble til eksterne kunnskapskilder.

Review-dybden skaleres nå etter kompleksiteten i endringen. Enkle endringer får rask sjekk, komplekse endringer får dypere analyse med verktøybruk.

**Kilde:** [Shape Copilot code review around your team](https://github.blog/changelog/2026-06-02-shape-copilot-code-review-around-your-team) (GitHub Changelog, 2. juni 2026)

## 6. Chronicle: innsikt på tvers av agentsesjoner

Ny `/chronicle`-kommando gjør alle Copilot-sesjoner søkbare. Chronicle bygger en historikk av alt agenten gjør — feilsøking, code reviews, feature-utvikling — og lar deg spørre mot den.

Bruksområder: standup-oppsummeringer, personlige produktivitetsmetrikker, og gjenfinning av tidligere tilnærminger til lignende problemer.

**Kilde:** [Gain insights across your agent sessions with Chronicle](https://github.blog/changelog/2026-06-02-gain-insights-across-your-agent-sessions-with-chronicle) (GitHub Changelog, 2. juni 2026)

## 7. Agent apps på GitHub Marketplace

GitHub lanserer agent apps — tredjeparts AI-agenter som installeres fra Marketplace og integreres direkte i GitHub. Du installerer en agent app som en vanlig GitHub App, og den kan delta i workflows, svare på issues, eller utføre spesialiserte oppgaver.

Dette åpner for et økosystem av spesialiserte agenter fra partnere — sikkerhetsskanning, compliance-sjekk, dokumentasjonsgenerering og mer.

**Kilde:** [Extend GitHub with agent apps](https://github.blog/changelog/2026-06-02-extend-github-with-agent-apps) (GitHub Changelog, 2. juni 2026)

## 8. Copilot CLI: nytt UI, prompt-planlegging og stemme

Copilot CLI får en stor oppgradering ved Build 2026. Rubber duck, prompt-planlegging (/every og /after) og stemmeinput er nå GA. I tillegg er et nytt eksperimentelt terminal-grensesnitt med tabs under utvikling.

Med `/every` kan du planlegge tilbakevendende oppgaver (f.eks. «sjekk security advisories hver mandag»). Med `/after` kan du kjede oppgaver som trigges etter at en annen er ferdig. Stemmeinput bruker lokal tale-til-tekst — ingen lyd forlater maskinen.

**Kilde:** [Copilot CLI: Improved UI, rubber duck, prompt scheduling, and voice input](https://github.blog/changelog/2026-06-02-copilot-cli-improved-ui-rubber-duck-prompt-scheduling-and-voice-input) (GitHub Changelog, 2. juni 2026)

## 9. Nye modeller: Gemini, MAI-Code-1-Flash og GPT-4.1-avvikling

Tre modellendringer på én dag. Gemini 3.1 Pro (preview) og Gemini 3.5 Flash er nå tilgjengelig i Copilot CLI, cloud agent og Copilot-appen. MAI-Code-1-Flash, Microsofts nye lille kodemodell, rulles ut i VS Code med «best-in-class quality for its size». Samtidig er GPT-4.1 offisielt avviklet fra 1. juni.

Modellutvalget i Copilot fortsetter å vokse. Utviklere kan nå velge mellom OpenAI, Anthropic, Google og Microsoft-modeller avhengig av oppgavetype.

**Kilde:** [Gemini models in Copilot CLI, cloud agent, and the Copilot app](https://github.blog/changelog/2026-06-02-gemini-models-in-copilot-cli-cloud-agent-and-the-copilot-app), [MAI-Code-1-Flash is now available for GitHub Copilot](https://github.blog/changelog/2026-06-02-mai-code-1-flash-is-now-available-for-github-copilot), [GPT-4.1 deprecated](https://github.blog/changelog/2026-06-02-gpt-4-1-deprecated) (GitHub Changelog, 2. juni 2026)

Copilot Memory med brukerpreferanser rulles ut til Business og Enterprise i public preview. Admins får nye kontroller for å styre hva Memory kan lagre og hvem som kan bruke det.

Etter utrullingen til Pro/Pro+-brukere tidligere kommer nå enterprise-versjonen med organisasjonsnivå-policyer. Dette lar Nav-admins aktivere Memory kontrollert, med innsyn i hva som lagres.

**Kilde:** [Copilot Memory supports user preferences for Business & Enterprise](https://github.blog/changelog/2026-06-02-copilot-memory-supports-user-preferences-for-business-enterprise) (GitHub Changelog, 2. juni 2026)

## 11. Kontekstvinduer på én million tokens og konfigurerbare resonneringsnivåer

GitHub Copilot støtter nå kontekstvinduer på opptil én million tokens. Det betyr at agenten kan jobbe med store kodebaser, lange dokumenter og komplekse flerfilsprosjekter uten å miste kontekst underveis. Funksjonen er tilgjengelig i VS Code, Copilot CLI og Copilot-appen.

I tillegg kan utviklere nå velge resonneringsnivå — fra raske, lette svar til dyp, utvidet tenkning for arkitekturproblemer og vanskelig feilsøking. Høyere resonneringsnivå og større kontekstvinduer bruker flere AI-credits per interaksjon, så GitHub anbefaler standardinnstillingene for daglige oppgaver.

**Kilde:** [Larger context windows and configurable reasoning levels for GitHub Copilot](https://github.blog/changelog/2026-06-04-larger-context-windows-and-configurable-reasoning-levels-for-github-copilot) (GitHub Changelog, 4. juni 2026)

## 12. Agent tasks REST API

Copilot Pro, Pro+ og Max-brukere kan nå programmatisk starte og spore cloud agent-oppgaver via en ny REST API i public preview. Cloud agent jobber i bakgrunnen i sitt eget utviklingsmiljø der den gjør kodeendringer, validerer dem og åpner en pull request.

API-et gjør det enkelt å integrere cloud agent i egne automatiseringer. Eksempler: fane ut refaktoreringer eller migrasjoner over mange repoer fra et enkelt skript, sette opp nye repoer fra en intern utviklerportal, eller automatisk forberede ukentlige releaser med release notes. Autentisering støtter PAT (klassisk og fine-grained) og OAuth-tokens.

**Kilde:** [Agent tasks REST API now available for Copilot Pro, Pro+, and Max](https://github.blog/changelog/2026-06-04-agent-tasks-rest-api-now-available-for-copilot-pro-pro-and-max) (GitHub Changelog, 4. juni 2026)

## 13. VS Code mai-utgivelser: Agents-vindu og remote agents

VS Code v1.120–v1.123 bringer flere store nyheter. Agents-vinduet er nå tilgjengelig i Stable som preview — en agent-first-overflate fokusert på oppgavefullføring fremfor redigering. Remote agents (preview) lar deg kjøre sesjoner på fjernmaskiner over SSH eller Dev Tunnels, og sesjonene fortsetter selv når klienten kobles fra.

GitHub investerer videre i Agent Host Protocol (AHP) — en åpen protokoll for synkronisering av agentsesjon-tilstand på tvers av flere klienter. BYOK (bring-your-own-key) er utvidet til lukkede nettverk (air-gapped environments), og utviklere kan nå styre hvilke modeller som håndterer verktøyoppgaver som commit-meldinger.

**Kilde:** [GitHub Copilot in Visual Studio Code, May releases](https://github.blog/changelog/2026-06-03-github-copilot-in-visual-studio-code-may-releases) (GitHub Changelog, 3. juni 2026)

## 14. Copilot Chat med PR-kontekst er generelt tilgjengelig

Copilot Chat på github.com med rikere kontekst for pull requests går fra public preview til generelt tilgjengelig. Funksjonen lar deg se samtalen direkte ved siden av PR-diffen, stille spørsmål om endringer, og få oppsummeringer — uten å veksle mellom vinduer.

Chatten forstår nå hele PR-konteksten: kommentarer, filendringer, commits og reviews. GitHub har også forbedret ytelsen slik at du får kvalitetssvar raskere.

**Kilde:** [Copilot Chat brings richer context to pull requests](https://github.blog/changelog/2026-06-04-copilot-chat-brings-richer-context-to-pull-requests) (GitHub Changelog, 4. juni 2026)
