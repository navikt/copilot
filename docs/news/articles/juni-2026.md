Microsoft Build 2026 ga en omfattende oppdatering for Copilot 2. juni. Hovedtemaene er tydelige: agentbasert utvikling modnes med sandkasser og automatiseringer, SDK-et blir allment tilgjengelig (GA) for egne integrasjoner, Copilot-appen får «canvases» som et nytt interaksjonsmønster, og kodevurderinger (code reviews) kan tilpasses med MCP og egne ferdigheter (skills). For Nav betyr dette at verktøyene for virksomhetsstyrt AI-utvikling tar et stort steg fremover.

---

## 1. Copilot SDK er generelt tilgjengelig

GitHub Copilot SDK har gått fra forhåndsversjon (public preview) til generell tilgjengelighet (GA). SDK-et gir programmatisk tilgang til det samme kjøretidsmiljøet (runtimen) for agenter som driver Copilot — inkludert planlegging, verktøykall, filendringer og samtalesesjoner over flere steg (multi-turn).

SDK-et er nå tilgjengelig på seks språk: TypeScript, Python, Go, .NET, Rust (ny) og Java (ny). Nøkkelfunksjonene inkluderer støtte for egendefinerte verktøy (custom tools) og MCP-integrasjon, finjustering av systemprompten uten å måtte skrive den fra bunnen av, OpenTelemetry-sporing med W3C-sporkontekst, samt fleksibel autentisering via GitHub OAuth, GitHub Apps, miljøvariabel-tokens og BYOK (Bring Your Own Key).

Siden forhåndsversjonen har SDK-et blitt brukt til å bygge CI/CD-assistenter, interne utviklerverktøy og brukerrettede AI-funksjoner. API-grensesnittet er nå stabilt og produksjonsklart.

**Kilde:** [Copilot SDK is now generally available](https://github.blog/changelog/2026-06-02-copilot-sdk-is-now-generally-available) (GitHub Changelog, 2. juni 2026)

## 2. Sky- og lokale sandkasser i public preview

Copilot kan nå kjøre verktøy i isolerte sandkasser — både lokalt på utviklerens maskin og i skyen. Dette sikrer at agentens verktøykjøring skjer i et lukket miljø, uten direkte tilgang til vertssystemet.

Lokale sandkasser isolerer filsystemet og prosesser under arbeid. Skysandkasser (cloud sandboxes) gir fullverdige utviklingsmiljøer i skyen. Begge variantene er utviklet for å gjøre agentbasert utvikling tryggere, spesielt når agenten installerer avhengigheter, kjører tester eller gjør endringer i filer.

**Kilde:** [Cloud and local sandboxes for GitHub Copilot now in public preview](https://github.blog/changelog/2026-06-02-cloud-and-local-sandboxes-for-github-copilot-now-in-public-preview) (GitHub Changelog, 2. juni 2026)

## 3. Copilot-appen: canvases og utvidet tilgang

Copilot-appen er nå tilgjengelig for alle Pro-, Pro+-, Business- og Enterprise-kunder. Den største nyheten i denne utgivelsen er «canvases» — interaktive arbeidsflater der utviklere og agenter kan samarbeide.

Et canvas er en strukturert og interaktiv flate for et spesifikt arbeidsobjekt — som en plan, en pull request (PR), en nettlesersesjon, en terminal eller en sjekkliste. Tre parter deler et canvas: brukeren som inspiserer og styrer, agenten som oppdaterer og bruker flaten til å dokumentere fremdrift, og selve applikasjonen som kobler alt sammen til underliggende artefakter. GitHub refererer til dette som «agent experience» (AX).

Blant øvrige nyheter finner vi stemmesamtaler med lokal tale-til-tekst, skysesjoner direkte fra appen, tidsplanlagte automatiseringer, agentbasert nettlesing og Copilot CLI-sesjoner som er synlige under «My work»-visningen i appen.

**Kilde:** [Expanded technical preview availability for the GitHub Copilot app](https://github.blog/changelog/2026-06-02-expanded-technical-preview-availability-for-the-github-copilot-app) (GitHub Changelog, 2. juni 2026)

## 4. Planlagte automatiseringer for cloud agent

Skyagenten (cloud agent) kan nå kjøre automatisk — enten etter en tidsplan eller som respons på hendelser i kodeforvaret (repository-hendelser). Med disse automatiseringene kan du overlate repetitive oppgaver til agenten uten at din egen maskin må være påslått.

Eksempler på bruksområder er daglige oppdateringer av avhengigheter (dependencies), automatisk triagering av nye saker (issues), periodisk kodeopprydding eller umiddelbare reaksjoner på hendelser som nye PR-er eller byggfeil.

**Kilde:** [Schedule and automate tasks with Copilot cloud agent](https://github.blog/changelog/2026-06-02-schedule-and-automate-tasks-with-copilot-cloud-agent) (GitHub Changelog, 2. juni 2026)

## 5. Code review med agent skills og MCP

Copilot code review kan nå skreddersys til teamets egne verktøy og standarder. To nye funksjoner er i public preview: Agentferdigheter (agent skills) lar kodevurderingen kjøre teamets egne verktøy (som lintere, typekontrollere og sikkerhetsskannere) underveis, og MCP-integrasjon (Model Context Protocol) gjør det mulig å koble til eksterne kunnskapskilder.

Dybden på kodevurderingen skaleres automatisk etter endringens kompleksitet. Enkle endringer får en rask sjekk, mens komplekse endringer går gjennom en dypere analyse med aktiv verktøybruk.

**Kilde:** [Shape Copilot code review around your team](https://github.blog/changelog/2026-06-02-shape-copilot-code-review-around-your-team) (GitHub Changelog, 2. juni 2026)

## 6. Chronicle: innsikt på tvers av agentsesjoner

Den nye kommandoen `/chronicle` gjør alle verktøysesjoner søkbare. Chronicle bygger opp en historikk over alt agenten foretar seg — inkludert feilsøking, kodevurderinger og funksjonsutvikling — og lar deg søke og stille spørsmål til historikken.

Aktuelle bruksområder er automatisk generering av standup-oppsummeringer, personlig produktivitetsstatistikk og gjenfinning av tidligere løsningsmetoder for lignende problemer.

**Kilde:** [Gain insights across your agent sessions with Chronicle](https://github.blog/changelog/2026-06-02-gain-insights-across-your-agent-sessions-with-chronicle) (GitHub Changelog, 2. juni 2026)

## 7. Agent-apper på GitHub Marketplace

GitHub lanserer agent-apper (agent apps) — tredjeparts AI-agenter som kan installeres fra GitHub Marketplace og integreres direkte i arbeidsflyten. Appene installeres på samme måte som en vanlig GitHub App, og kan deretter delta i workflows, svare på issues eller utføre spesialiserte oppgaver.

Dette oppretter et økosystem av spesialiserte agenter fra partnere innen områder som sikkerhetsskanning, etterlevelseskontroll (compliance) og automatisk dokumentasjonsgenerering.

**Kilde:** [Extend GitHub with agent apps](https://github.blog/changelog/2026-06-02-extend-github-with-agent-apps) (GitHub Changelog, 2. juni 2026)

## 8. Copilot CLI: nytt UI, prompt-planlegging og stemme

Copilot CLI får en stor oppgradering i forbindelse med Build 2026. Funksjoner som «rubber ducking», tidsstyrt planlegging av ledetekster (via `/every` og `/after`) og stemmeinput er nå generelt tilgjengelig (GA). I tillegg jobbes det med et nytt, eksperimentelt terminalgrensesnitt med faner (tabs).

Med `/every` kan du planlegge repeterende oppgaver (for eksempel «sjekk sikkerhetsvarsler hver mandag»). Med `/after` kan du kjede oppgaver slik at de starter automatisk når en annen er fullført. Stemmeinputen bruker lokal tale-til-tekst, slik at ingen lyddata forlater maskinen.

**Kilde:** [Copilot CLI: Improved UI, rubber duck, prompt scheduling, and voice input](https://github.blog/changelog/2026-06-02-copilot-cli-improved-ui-rubber-duck-prompt-scheduling-and-voice-input) (GitHub Changelog, 2. juni 2026)

## 10. Copilot Memory for Business og Enterprise

Copilot Memory, som lar verktøyet huske brukerpreferanser, rulles nå ut til Business og Enterprise i public preview. Administratorer får nye kontroller for å styre hva minnefunksjonen kan lagre, og hvem som har tilgang til å bruke den.

Etter at funksjonen tidligere ble rullet ut til Pro/Pro+-brukere, kommer nå enterprise-versjonen med retningslinjer på organisasjonsnivå. Dette gjør det mulig for Nav-administratorer å aktivere Copilot Memory under kontrollerte former, med godt innsyn i hva som lagres.

**Kilde:** [Copilot Memory supports user preferences for Business & Enterprise](https://github.blog/changelog/2026-06-02-copilot-memory-supports-user-preferences-for-business-enterprise) (GitHub Changelog, 2. juni 2026)

## 11. Kontekstvinduer på én million tokens og konfigurerbare resonneringsnivåer

GitHub Copilot støtter nå kontekstvinduer på opptil én million tokens. Det betyr at agenten kan arbeide med store kodebaser, omfattende dokumentasjon og komplekse prosjekter over flere filer uten å miste oversikten underveis. Funksjonen er tilgjengelig i VS Code, Copilot CLI og Copilot-appen.

I tillegg kan utviklere nå velge resonneringsnivå — fra raske, enkle svar til dyp, resonnerende tenkning for arkitekturspørsmål og avansert feilsøking. Høyere resonneringsnivå og større kontekstvinduer forbruker mer AI-credits per interaksjon, så GitHub anbefaler å bruke standardinnstillingene til daglige oppgaver.

**Kilde:** [Larger context windows and configurable reasoning levels for GitHub Copilot](https://github.blog/changelog/2026-06-04-larger-context-windows-and-configurable-reasoning-levels-for-github-copilot) (GitHub Changelog, 4. juni 2026)

## 12. REST-API for agentoppgaver (Agent tasks REST API)

Copilot Pro-, Pro+- og Max-brukere kan nå starte og spore oppgaver for skagenten (cloud agent) programmatisk via et nytt REST-API i public preview. Cloud-agenten jobber asynkront i et eget utviklingsmiljø der den gjør kodeendringer, validerer dem og oppretter en pull request.

API-et gjør det enkelt å integrere cloud-agenten i egne automatiseringsflyter. Aktuelle eksempler er refaktoreringer og migrasjoner som rulles ut over mange repositorier fra ett enkelt skript, oppsett av nye repositorier fra en intern utviklerportal, eller automatisk klargjøring av ukentlige versjonsutgivelser (releases) med tilhørende endringslogger. Autentiseringen støtter både personlige tilgangstokens (PAT) og OAuth-tokens.

**Kilde:** [Agent tasks REST API now available for Copilot Pro, Pro+, and Max](https://github.blog/changelog/2026-06-04-agent-tasks-rest-api-now-available-for-copilot-pro-pro-and-max) (GitHub Changelog, 4. juni 2026)

## 13. VS Code mai-utgivelser: Agents-vindu og remote agents

VS Code v1.120–v1.123 bringer flere store nyheter. Agent-vinduet (Agents) er nå tilgjengelig som en forhåndsvisning i stabil versjon (Stable) — en agentfokusert overflate innrettet mot oppgaveløsning fremfor ren koding. Eksterne agenter (remote agents) lar deg kjøre sesjoner på eksterne maskiner over SSH eller Dev Tunnels, og disse sesjonene fortsetter å kjøre selv om du kobler fra.

GitHub fortsetter å investere i Agent Host Protocol (AHP) — en åpen protokoll for å synkronisere tilstanden i agentsesjoner på tvers av ulike klienter. BYOK (Bring Your Own Key) er utvidet til lukkede nettverk (air-gapped environments), og utviklere kan nå selv styre hvilke modeller som skal håndtere spesifikke verktøyoppgaver, som for eksempel generering av commit-meldinger.

**Kilde:** [GitHub Copilot in Visual Studio Code, May releases](https://github.blog/changelog/2026-06-03-github-copilot-in-visual-studio-code-may-releases) (GitHub Changelog, 3. juni 2026)

## 14. Copilot Chat med PR-kontekst er generelt tilgjengelig

Copilot Chat på github.com med utvidet kontekst for pull requests (PR) går nå fra public preview til å bli allment tilgjengelig (GA). Funksjonen lar deg chatte direkte ved siden av PR-diffen, stille spørsmål om endringene og få raske oppsummeringer uten å måtte bytte vindu.

Chatten forstår hele konteksten rundt en PR, inkludert kommentarer, filendringer, commits og tidligere omtaler. GitHub har også forbedret ytelsen for å gi raskere og mer presise svar.

**Kilde:** [Copilot Chat brings richer context to pull requests](https://github.blog/changelog/2026-06-04-copilot-chat-brings-richer-context-to-pull-requests) (GitHub Changelog, 4. juni 2026)

## 16. VS Code 1.124: Autopilot, bakgrunnssesjoner og bedre navigasjon

VS Code 1.124 ble utgitt 10. juni med et sterkt fokus på agenter. Målet er å gjøre det enklere å jobbe med flere parallelle Copilot-sesjoner uten å miste oversikten eller konteksten.

Autopilot er nå aktivert som standard og avgjør selv når en oppgave er fullført. Du kan også starte nye sesjonsforespørsler i bakgrunnen, slik at du kan fortsette kodingen mens agenten arbeider. Dokumentasjonen for versjonsutgivelsen (release notes) trekker også frem enklere navigering mellom sesjoner, gjenoppretting ved omstart (restore on reload), nettleserhistorikk og sentralt styrte retningslinjer for utvidelser (enterprise-managed plugin policies).

## 17. VS Code-agenter får minne (memory) og Copilot Memory

VS Code dokumenterte 10. juni hvordan agenter bruker minne (memory) til å huske kontekst på tvers av samtaler. Minneverktøyet (Memory tool) er i forhåndsversjon og lagrer data lokalt på din maskin.

Dokumentasjonen deler minnet inn i tre nivåer: bruker (user), repositorium (repository) og sesjon (session). Brukerminne er tilpasset personlige preferanser, prosjektminne tar vare på prosjektspesifikke regler, og sesjonsminne holder på midlertidig kontekst. Copilot Memory er en separat, GitHub-driftet tjeneste som er prosjektspesifikk (repository-scoped), deles på tvers av ulike Copilot-flater, og verifiseres automatisk mot kodebasen før bruk. Lagrede opplysninger utløper etter 28 dager.

**Kilde:** [VS Code agents får memory og Copilot Memory](https://code.visualstudio.com/docs/agents/memory) (VS Code Documentation, 10. juni 2026)

## 18. Overgang til bruksbasert fakturering og AI-kreditter

Den lenge varslede overgangen til bruksbasert fakturering trådte i kraft 1. juni 2026. Den tidligere ordningen med flate abonnementspriser er erstattet med en modell der organisasjoner tildeles en månedlig pott med **GitHub AI Credits** (1 900 kreditter per bruker for Business-kunder, og 3 000 kreditter per bruker i en overgangsperiode ut august).

Forbruket måles basert på faktisk tokenforbruk (input, output og cachede tokens) etter spesifikke API-satser per modell. Standard kodefullføring (autocomplete) forblir ubegrenset og belaster ikke kredittbeholdningen, mens Copilot Chat, CLI-bruk og mer avanserte agentiske oppgaver forbruker kreditter.

**Kilde:** [Copilot går over til bruksbasert fakturering 1. juni](https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing/) (GitHub Blog, 27. april 2026)
