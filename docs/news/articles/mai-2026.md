Mai 2026 bringer sentralstyrt plugin-distribusjon for Copilot CLI, modellavviklinger med frist 1. juni, Rubber Duck cross-model i CLI, fire VS Code-releaser med /chronicle og BYOK, pausede individuelle planer, co-author-kontrovers og team-config for cplt.

## 1. Enterprise-managed plugins for Copilot CLI

Enterprise-admins kan nå distribuere plugins — custom agents, skills, hooks og MCP-konfigurasjoner — sentralt til alle Copilot CLI-brukere. Innstillingene defineres i `.github-private/.github/copilot/settings.json` og hentes automatisk for alle med Copilot Business eller Enterprise.

Dette gir Nav én sentral plass å styre verktøy fra, uten at utviklere må sette opp noe manuelt. Spesielt nyttig for å distribuere obligatoriske hooks og MCP-servere.

**Kilde:** [Enterprise-managed plugins in GitHub Copilot CLI are now in public preview](https://github.blog/changelog/2026-05-06-enterprise-managed-plugins-in-github-copilot-cli-are-now-in-public-preview/) (GitHub Changelog, 6. mai 2026)

## 2. GPT-5.2, GPT-5.2-Codex og Claude Sonnet 4 avvikles

Tre modeller fases ut i mai:

- **GPT-5.2 og GPT-5.2-Codex** fjernes fra alle Copilot-opplevelser 1. juni. Anbefalt erstatning: GPT-5.5 (generell) og GPT-5.3-Codex (kode).
- **Claude Sonnet 4** ble fjernet 7. mai. Anbefalt erstatning: Claude Sonnet 4.6.

Enterprise-admin bør sikre at erstatningsmodellene er aktivert i modellpolicyen. Modellene fjernes automatisk etter fristen.

**Kilder:**

- [Upcoming deprecation of GPT-5.2 and GPT-5.2-Codex](https://github.blog/changelog/2026-05-01-upcoming-deprecation-of-gpt-5-2-and-gpt-5-2-codex) (GitHub Changelog, 1. mai 2026)
- [Claude Sonnet 4 deprecated](https://github.blog/changelog/2026-05-07-claude-sonnet-4-deprecated/) (GitHub Changelog, 7. mai 2026)

## 3. Co-author-kontroversen i VS Code

VS Code 1.118 skrudde på `git.addAICoAuthor` som standard — en `Co-authored-by: Copilot`-trailer ble lagt til alle commits, også der Copilot ikke hadde bidratt. Utviklere reagerte med 372 tomler ned på PR-en. Microsoft reverserer i VS Code 1.119.

Sjekk VS Code-innstillinger etter oppgradering. Team med strenge regler for commit-metadata bør sette `git.addAICoAuthor` til `"off"` i workspace-settings.

**Kilder:**

- [VS Code Now Stamps GitHub Copilot as Git Commit Co-Author](https://winbuzzer.com/2026/05/03/vs-code-1-118-copilot-co-author-default-commits-xcxwbn/) (WinBuzzer, 3. mai 2026)
- [Microsoft fixes VS Code after Copilot credited human code](https://www.theregister.com/software/2026/05/04/microsoft-fixes-vs-code-after-copilot-credited-human-code/5223936) (The Register, 4. mai 2026)

## 4. Rubber Duck i Copilot CLI støtter flere modeller

Rubber Duck-agenten i Copilot CLI bruker nå Claude som kritiker i GPT-sesjoner og GPT-5.5 som kritiker i Claude-sesjoner. Det gir second opinion på tvers av modellfamilier — nyttig for code review der du vil ha et motvekt-perspektiv.

## 6. GitHub pauser registrering for individuelle planer

Nye registreringer for Copilot Pro, Pro+ og Student er midlertidig pauset. Eksisterende abonnenter beholder tilgangen, men Opus-modeller flyttes til Pro+. Business- og Enterprise-planer er ikke berørt.

## 7. cplt — team-config og `cplt init`

cplt støtter nå `.cplt.toml` — en konfigurasjonsfil du committer til repoet. Alle som kjører cplt i repoet får automatisk riktig sandbox-config.

Nytt: `cplt init` skanner prosjektet for byggsystemer og rammeverk, og genererer en `.cplt.toml` med riktige tillatelser. 15 detektorer dekker JVM, Node.js, Docker, Python, Rust, Go, Spring Boot, Ktor, Next.js, Flyway og mer. `cplt init --global` foreslår personlig config basert på installerte verktøy.

**Kilde:** [cplt init PR #42](https://github.com/navikt/cplt/pull/42) (cplt, mai 2026)

## 8. GitHub Copilot-appen i technical preview

GitHub lanserer en dedikert desktopapp for Copilot. Appen er en GitHub-native opplevelse for agentisk utvikling — du starter oppgaver direkte fra issues, PR-er eller kode, holder dem isolert i egne miljøer, og styrer dem fra ett samlet grensesnitt.

Dette er første gang Copilot lever utenfor en IDE som en frittstående app. Målet er å gjøre det enklere å starte og overvåke agentiske oppgaver uten å forstyrre det du allerede jobber med i editoren.

**Kilde:** [GitHub Copilot app is now available in technical preview](https://github.blog/changelog/2026-05-14-github-copilot-app-is-now-available-in-technical-preview) (GitHub Changelog, 14. mai 2026)

## 10. Cloud agent støtter automatisk modellvalg

Copilot cloud agent støtter nå «Auto» i modellvelgeren. Når du velger Auto, velger Copilot intelligent den beste tilgjengelige modellen basert på systemhelse og modellkapasitet. Dette gir bedre tilgjengelighet og potensielt lavere kostnad uten at du trenger å velge modell manuelt.

**Kilde:** [Copilot cloud agent supports auto model selection](https://github.blog/changelog/2026-05-14-copilot-cloud-agent-supports-auto-model-selection) (GitHub Changelog, 14. mai 2026)

## 11. GPT-5.3-Codex er nå standardmodell for Business og Enterprise

GPT-5.3-Codex erstatter GPT-4.1 som standardmodell («base model») for alle Copilot Business- og Enterprise-organisasjoner. Standardmodellen brukes når organisasjonen ikke har godkjent andre modeller gjennom intern modellpolicy.

GPT-5.3-Codex er også GitHubs første LTS-modell (Long-Term Support) i samarbeid med OpenAI — garantert tilgjengelig til 4. februar 2027. GitHub rapporterer høy «code survival rate» blant enterprise-kunder. Modellen har 1× premium request unit-multiplikator. GPT-4.1 forblir tilgjengelig midlertidig med 0× multiplikator, men avvikles ved overgang til bruksbasert fakturering 1. juni.

**Kilde:** [GPT-5.3-Codex is now the base model for Copilot Business and Enterprise](https://github.blog/changelog/2026-05-17-gpt-5-3-codex-is-now-the-base-model-for-copilot-business-and-enterprise) (GitHub Changelog, 17. mai 2026)

## 12. Grok Code Fast 1 fjernet fra Copilot

Grok Code Fast 1 er deprecated og fjernet fra alle Copilot-opplevelser (Chat, inline edits, ask/agent mode og code completions) per 15. mai. Administratorer bør sikre at alternative modeller er aktivert i modellpolicyen — brukere som hadde Grok som foretrukket modell vil automatisk falle tilbake til standardmodellen.

**Kilde:** [Grok Code Fast 1 deprecated](https://github.blog/changelog/2026-05-15-grok-code-fast-1-deprecated) (GitHub Changelog, 15. mai 2026)

## 13. Copilot Memory støtter personlige preferanser

Copilot Memory utvides med bruker-nivå preferanser i early access for Pro og Pro+-brukere. Tidligere lagret Memory kun informasjon på repo-nivå — nå kan Copilot også lagre personlige preferanser om kommunikasjonsstil, tone og arbeidsmåte. Preferansene følger brukeren på tvers av alle repoer og Copilot-agenter uten å påvirke andre i samme repo.

Funksjonen er foreløpig begrenset til individuelle planer, men GitHub planlegger å utvide til flere planer. For Nav er dette en peker på hvor Memory-funksjonaliteten er på vei — personlige preferanser som overlever på tvers av sesjoner og verktøy.

**Kilde:** [Copilot Memory supports user preferences for Pro, Pro+ users](https://github.blog/changelog/2026-05-15-copilot-memory-supports-user-preferences-for-pro-pro-users) (GitHub Changelog, 15. mai 2026)

## 14. Ett-klikks fiks for feilende Actions

Når en GitHub Actions-jobb feiler, kan Copilot Business- og Enterprise-brukere nå klikke «Fix with Copilot» direkte på workflow-loggsiden. Cloud agent undersøker feilen, pusher en fiks til branchen din og tagger deg for review. Nyttig for tidkrevende men enkle feil som testfeil eller linter-brudd.

Organisasjonen må ha cloud agent aktivert. Funksjonaliteten kjører i cloud agents eget miljø — ingen lokal oppsett kreves.

**Kilde:** [One-click fixes for failing Actions with Copilot cloud agent](https://github.blog/changelog/2026-05-18-one-click-fixes-for-failing-actions-with-copilot-cloud-agent) (GitHub Changelog, 18. mai 2026)

## 17. Remote control for CLI-sesjoner er GA

Remote control for Copilot CLI-sesjoner er nå generelt tilgjengelig på GitHub Mobile, github.com, VS Code og JetBrains. Du kan starte en sesjon i terminalen og overvåke eller styre den fra en annen enhet i sanntid. Nytt i GA: støtte for ikke-GitHub-repoer og kataloger uten tilknyttet repo.

**Kilde:** [Remote control for Copilot CLI sessions now generally available on mobile, web, and VS Code](https://github.blog/changelog/2026-05-18-remote-control-for-copilot-cli-sessions-now-generally-available-on-mobile-web-and-vs-code) (GitHub Changelog, 18. mai 2026)

Nytt REST API-endepunkt (public preview) returnerer en repos cloud agent-konfigurasjon: MCP-servere, aktiverte verktøy, Actions workflow-policy og brannmurregler. Gjør det enkelt å verifisere sikkerhetsposituren på tvers av mange repoer programmatisk.

## 19. Copilot-chat på github.com åpner i kontekst

Copilot-chat åpner nå som et panel på siden du ser på i stedet for å navigere bort. Issues, PR-er og repoer legges automatisk til som kontekst. Du kan også konvertere en samtale til en agent-sesjon ved å be Copilot lage en PR. GA for alle Copilot-planer.

## 20. Gemini 3.5 Flash er generelt tilgjengelig

Googles nyeste Flash-modell ruller ut i GitHub Copilot. Gemini 3.5 Flash gir nær Pro-kvalitet til Flash-hastighet med sterk verktøybruk og høy cache-effektivitet — godt egnet for raske, iterative agentiske workflows. Modellen har 14× premium request-multiplikator (tentativ). Tilgjengelig for Pro, Pro+, Business og Enterprise etter at admin aktiverer den i modellpolicyen.

**Kilde:** [Gemini 3.5 Flash is generally available for GitHub Copilot](https://github.blog/changelog/2026-05-19-gemini-3-5-flash-is-generally-available-for-github-copilot) (GitHub Changelog, 19. mai 2026)

## 21. Fiks code review-tilbakemeldinger med cloud agent

«Implement suggestion»-knappen i code review er erstattet av «Fix with Copilot» med en dialog som gir deg kontroll over hva som skjer. Du kan velge å fikse i samme branch, lage ny branch, eller batche flere kommentarer med «Fix batch with Copilot» fra PR-oversikten. Kobler code review tettere til cloud agent.

**Kilde:** [Easily apply Copilot code review feedback with Copilot cloud agent](https://github.blog/changelog/2026-05-19-easily-apply-copilot-code-review-feedback-with-copilot-cloud-agent) (GitHub Changelog, 19. mai 2026)
