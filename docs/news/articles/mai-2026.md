Mai 2026 bringer sentralstyrt plugin-distribusjon for Copilot CLI, modellavviklinger med frist 1. juni, Rubber Duck cross-model i CLI, fire VS Code-releaser med /chronicle og BYOK, pausede individuelle planer, co-author-kontrovers og team-config for cplt.

## 1. Enterprise-managed plugins for Copilot CLI

Enterprise-admins kan nå distribuere plugins — custom agents, skills, hooks og MCP-konfigurasjoner — sentralt til alle Copilot CLI-brukere. Innstillingene defineres i `.github-private/.github/copilot/settings.json` og hentes automatisk for alle med Copilot Business eller Enterprise.

Dette gir Nav én sentral plass å styre verktøy fra, uten at utviklere må sette opp noe manuelt. Spesielt nyttig for å distribuere obligatoriske hooks og MCP-servere.

**Kilde:** [Enterprise-managed plugins in GitHub Copilot CLI are now in public preview](https://github.blog/changelog/2026-05-06-enterprise-managed-plugins-in-github-copilot-cli-are-now-in-public-preview/) (GitHub Changelog, 6. mai 2026)

## 2. GPT-5.2, GPT-5.2-Codex og Claude Sonnet 4 avvikles

Tre modeller fases ut i mai:

- **GPT-5.2 og GPT-5.2-Codex** fjernes fra alle Copilot-opplevelser 1. juni. Anbefalt erstatning: GPT-5.5 (generell) og GPT-5.3-Codex (kode).
- **Claude Sonnet 4** ble fjernet 7. mai. Anbefalt erstatning: Claude Sonnet 4.6.

Enterprise-admin bør sikre at erstatningsmodellene er aktivert i modellpolicyen. Modellene fjernes automatisk etter fristen.

**Kilder:**

- [Upcoming deprecation of GPT-5.2 and GPT-5.2-Codex](https://github.blog/changelog/2026-05-01-upcoming-deprecation-of-gpt-5-2-and-gpt-5-2-codex) (GitHub Changelog, 1. mai 2026)
- [Claude Sonnet 4 deprecated](https://github.blog/changelog/2026-05-07-claude-sonnet-4-deprecated/) (GitHub Changelog, 7. mai 2026)

## 3. Co-author-kontroversen i VS Code

VS Code 1.118 skrudde på `git.addAICoAuthor` som standard — en `Co-authored-by: Copilot`-trailer ble lagt til alle commits, også der Copilot ikke hadde bidratt. Utviklere reagerte med 372 tomler ned på PR-en. Microsoft reverserer i VS Code 1.119.

Sjekk VS Code-innstillinger etter oppgradering. Team med strenge regler for commit-metadata bør sette `git.addAICoAuthor` til `"off"` i workspace-settings.

**Kilder:**

- [VS Code Now Stamps GitHub Copilot as Git Commit Co-Author](https://winbuzzer.com/2026/05/03/vs-code-1-118-copilot-co-author-default-commits-xcxwbn/) (WinBuzzer, 3. mai 2026)
- [Microsoft fixes VS Code after Copilot credited human code](https://www.theregister.com/software/2026/05/04/microsoft-fixes-vs-code-after-copilot-credited-human-code/5223936) (The Register, 4. mai 2026)

## 4. Rubber Duck i Copilot CLI støtter flere modeller

Rubber Duck-agenten i Copilot CLI bruker nå Claude som kritiker i GPT-sesjoner og GPT-5.5 som kritiker i Claude-sesjoner. Det gir second opinion på tvers av modellfamilier — nyttig for code review der du vil ha et motvekt-perspektiv.

**Kilde:** [Rubber Duck in GitHub Copilot CLI now supports more models](https://github.blog/changelog/2026-05-07-rubber-duck-in-github-copilot-cli-now-supports-more-models/) (GitHub Changelog, 7. mai 2026)

## 5. VS Code april-releaser — /chronicle, BYOK og nettleserdeling

VS Code v1.116–v1.119 ga fire releaser med viktige Copilot-endringer:

- **/chronicle** — søkbar chat-historikk på tvers av sesjoner
- **BYOK (bring-your-own-key)** — Business/Enterprise-brukere kan bruke egne API-nøkler i editoren
- **Terminal- og nettlesertilgang** — agenter kan lese og skrive i åpne terminaler og dele nettleserfaner

**Kilde:** [GitHub Copilot in Visual Studio Code — April releases](https://github.blog/changelog/2026-05-06-github-copilot-in-visual-studio-code-april-releases/) (GitHub Changelog, 6. mai 2026)

## 6. GitHub pauser registrering for individuelle planer

Nye registreringer for Copilot Pro, Pro+ og Student er midlertidig pauset. Eksisterende abonnenter beholder tilgangen, men Opus-modeller flyttes til Pro+. Business- og Enterprise-planer er ikke berørt.

For Nav betyr dette lite direkte (vi bruker Enterprise), men det signaliserer at GitHub konsoliderer prismodellen.

**Kilde:** [Changes to GitHub Copilot individual plans](https://github.blog/news-insights/company-news/changes-to-github-copilot-individual-plans/) (GitHub Blog, 20. april 2026)

## 7. cplt — team-config og `cplt init`

cplt støtter nå `.cplt.toml` — en konfigurasjonsfil du committer til repoet. Alle som kjører cplt i repoet får automatisk riktig sandbox-config.

Nytt: `cplt init` skanner prosjektet for byggsystemer og rammeverk, og genererer en `.cplt.toml` med riktige tillatelser. 15 detektorer dekker JVM, Node.js, Docker, Python, Rust, Go, Spring Boot, Ktor, Next.js, Flyway og mer. `cplt init --global` foreslår personlig config basert på installerte verktøy.

**Kilde:** [cplt init PR #42](https://github.com/navikt/cplt/pull/42) (cplt, mai 2026)

## 8. GitHub Copilot-appen i technical preview

GitHub lanserer en dedikert desktopapp for Copilot. Appen er en GitHub-native opplevelse for agentisk utvikling — du starter oppgaver direkte fra issues, PR-er eller kode, holder dem isolert i egne miljøer, og styrer dem fra ett samlet grensesnitt.

Dette er første gang Copilot lever utenfor en IDE som en frittstående app. Målet er å gjøre det enklere å starte og overvåke agentiske oppgaver uten å forstyrre det du allerede jobber med i editoren.

**Kilde:** [GitHub Copilot app is now available in technical preview](https://github.blog/changelog/2026-05-14-github-copilot-app-is-now-available-in-technical-preview) (GitHub Changelog, 14. mai 2026)

## 9. Teambaserte bruksmetrikker via API

Copilot usage metrics API har fått et nytt user-teams-endepunkt som mapper hver lisensiert bruker til teamene de tilhører. Ved å joine dette med eksisterende bruksdata kan organisasjoner bryte ned Copilot-bruk per team — nyttig for å forstå adopsjon og planlegge AI Credits-budsjetter.

**Kilde:** [Team-level Copilot usage metrics now available via API](https://github.blog/changelog/2026-05-14-team-level-copilot-usage-metrics-now-available-via-api) (GitHub Changelog, 14. mai 2026)

## 10. Cloud agent støtter automatisk modellvalg

Copilot cloud agent støtter nå «Auto» i modellvelgeren. Når du velger Auto, velger Copilot intelligent den beste tilgjengelige modellen basert på systemhelse og modellkapasitet. Dette gir bedre tilgjengelighet og potensielt lavere kostnad uten at du trenger å velge modell manuelt.

**Kilde:** [Copilot cloud agent supports auto model selection](https://github.blog/changelog/2026-05-14-copilot-cloud-agent-supports-auto-model-selection) (GitHub Changelog, 14. mai 2026)
