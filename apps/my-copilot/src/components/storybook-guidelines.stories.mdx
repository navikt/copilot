# Storybook-guidelines for videokomponenter

Denne siden beskriver anbefalt struktur og kultur for stories i `my-copilot`.

## Struktur

Bruk følgende hierarki:

- `Video/Primitives/*` for små byggesteiner
- `Video/Controls/*` for interaktive kontroller
- `Video/Panels/*` for informasjonsflater
- `Video/HUD/*` for sammensatt overlay-lag
- `Video/Pages/*` for full side-/feature-komposisjon

## Navngivning

- Story-titler skal være konkrete (`Paused`, `Playing`, `Completed`)
- Beskrivende `docs.description.component` skal forklare **hva** komponenten brukes til
- Story-beskrivelse skal forklare **hvilken tilstand** som demonstreres

## Datagrunnlag

- Bruk realistiske fixtures fra dev-bucket når mulig
- Hold fallback-fixtures stabile for deterministisk UI-testing
- Unngå skjøre stories som krever live backend for å rendre

## Tilgjengelighet og designkvalitet

- Verifiser tastaturfokus i alle interaktive stories
- Bruk mørk bakgrunn for videokomponenter for korrekt kontrastvurdering
- Bruk Aksel spacing-tokens via `Box`/designsystem der spacing må styres

## Kultur for review

Ved nye videokomponenter:

1. Legg til minst én story for normaltilstand
2. Legg til minst én story for edge state (tom, lang tekst, completed, etc.)
3. Legg inn kort doc-beskrivelse av forventet bruk
4. Kjør `pnpm build-storybook` før PR
